# 2026-03-08 변경사항

## 작업 목적

기존 앱은 기능은 충분했지만, 팀원이 처음 접속했을 때 어떤 화면에서 협업하고, 어떤 화면에서 관리하고, 어떤 화면에서 보고해야 하는지가 즉시 드러나지 않았습니다.

이번 변경은 기존 `업무 / 프로젝트 / 체크리스트 / 회의록` 기능을 유지한 채, 팀 운영의 시작점을 `대시보드`로 재구성하는 데 목적이 있습니다.

## 이번에 변경한 내용

### 1. 팀 운영 대시보드 추가

- 새 `대시보드` 탭을 추가했습니다.
- 앱 기본 진입 화면을 `dashboard`로 변경했습니다.
- 대시보드에서 아래 3가지 흐름으로 바로 이동할 수 있게 구성했습니다.
  - 협업: 업무/회의록 중심
  - 관리: 프로젝트/체크리스트 중심
  - 보고: 당일 현황 요약 복사 중심

### 2. 팀이 바로 봐야 할 핵심 지표 정리

대시보드 상단에서 아래 수치를 즉시 볼 수 있게 했습니다.

- 활성 업무 수
- 지연 업무 수
- 오늘 마감 업무 수
- 체크리스트 미할당 수

이 값들은 기존 `tasks`, `projects`, `checklistRows` 데이터를 재사용해서 계산하도록 구성했습니다.

### 3. 협업용 우선 확인 영역 추가

`오늘 바로 볼 업무` 영역을 추가했습니다.

- 지연 업무
- 오늘 마감 업무
- 긴급 표시 업무
- 담당 미지정 업무

위 조건을 기준으로 우선순위를 정렬해 바로 업무 상세로 들어갈 수 있게 했습니다.

### 4. 관리용 프로젝트/체크리스트 포커스 추가

관리자가 바로 점검할 수 있도록 아래 영역을 추가했습니다.

- `이번 주 프로젝트`
  - 행사일이 가까운 프로젝트를 우선 노출
- `체크리스트 할당 포커스`
  - 현재 선택된 행사 기준 미할당 항목 우선 노출

### 5. 보고용 요약 문구 복사 기능 추가

대시보드에서 당일 기준 보고 문구를 자동 생성하도록 했습니다.

포함 내용:

- 프로젝트 건수
- 활성 업무 건수
- 오늘 마감 건수
- 지연 건수
- 체크리스트 미할당 건수
- 이번 주 주요 일정
- 우선 확인 업무

또한 브라우저 클립보드 복사 기능을 연결하고, 성공/실패 토스트를 표시하도록 했습니다.

### 6. 기존 화면과의 연결 정리

- 사이드바에 `대시보드` 메뉴를 추가했습니다.
- `사용법` 탭에도 대시보드 설명과 진입 버튼을 추가했습니다.
- 기존 `프로젝트 / 업무 / 체크리스트 / 회의록 / 사용법` 흐름은 그대로 유지했습니다.

### 7. 타입 정리

공용 타입의 `TopView` 정의가 실제 앱 상태와 어긋나 있던 부분을 정리했습니다.

- `dashboard`
- `meetings`

를 공용 타입에도 반영했습니다.

## 기대 효과

- 새 팀원이 접속했을 때 시작 위치가 명확해집니다.
- 협업, 운영 관리, 보고가 각각 어디서 이뤄지는지 바로 이해할 수 있습니다.
- 기존 데이터를 버리지 않고 실제 운영 흐름만 더 읽기 쉽게 정리할 수 있습니다.
- 보고용 문구를 별도 정리하지 않고 바로 복사해서 공유할 수 있습니다.
