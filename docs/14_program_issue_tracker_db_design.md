# 프로그램 이슈 트래커 DB 설계

## 목적
- 팀 내부 사용자가 웹 앱 사용 중 발견한 버그, 개선 요청, 질문, 제안을 기록한다.
- 기존 `업무 피드백` DB와 분리해 프로그램 자체 이슈만 관리한다.

## Notion DB
- env key: `NOTION_PROGRAM_ISSUES_DB_ID`
- 권장 DB 이름: `프로그램 이슈 트래커`

## 권장 속성
| 속성 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `이슈 제목` | `title` | yes | 한 줄 요약 제목 |
| `상세 내용` | `rich_text` | no | 문제 상황/개선 요청 상세 |
| `구분` | `select` | no | `버그`, `개선`, `질문`, `제안`, `기타` |
| `화면/기능` | `rich_text` | no | 예: `행사 체크리스트`, `NAS 가이드` |
| `중요도` | `select` | no | `낮음`, `보통`, `높음`, `긴급` |
| `상태` | `select` | no | `미해결`, `확인중`, `진행중`, `보류`, `해결` |
| `제보자` | `rich_text` | no | 이름 또는 팀명 |
| `담당자` | `rich_text` | no | 처리 담당자 |
| `보류 사유` | `rich_text` | no | `상태=보류`일 때 이유 기록 |
| `재현 방법` | `rich_text` | no | 재현 순서 또는 조건 |
| `비고` | `rich_text` | no | 기타 메모 |
| `등록일` | `date` | no | 최초 등록일 |
| `해결일` | `date` | no | 해결 완료일 |

## 상태 운용 규칙
- `미해결`: 새로 등록되었고 아직 확인 전인 상태
- `확인중`: 현상 파악/재현 확인 중
- `진행중`: 실제 수정 작업 진행 중
- `보류`: 당장 처리하지 않기로 결정한 상태
- `해결`: 수정 또는 정책 결정으로 종료된 상태

## 보류 규칙
- `상태`를 `보류`로 바꾸면 `보류 사유`를 함께 남긴다.
- 예: 우선순위 낮음, 정책 확정 대기, 외부 서비스 제약, 재현 불가
