# 06_업로드

## 1) 목적
- 웹페이지에서 작업 완성본 파일을 업로드하면, 서버(Cloudflare Worker)가 라우팅 로직에 따라 Google Drive와 Synology로 자동 업로드한다.
- 브라우저에는 외부 저장소 자격증명을 노출하지 않는다.

## 2) 범위
- In Scope
  - 업로드 요청/완료 API
  - 업로드 대상 라우팅(프로젝트/파일명 규칙 기반)
  - Google Drive 업로드
  - Synology 업로드
  - 업로드 상태 조회 및 실패 재시도
- Out of Scope (현재)
  - 사용자별 복잡 권한 모델
  - 대용량 전송 최적화(청크 업로드/멀티파트 고도화)

## 3) 기본 아키텍처
1. 브라우저 -> Worker로 업로드 시작 요청
2. Worker가 원본 저장소(R2)에 임시 저장 경로를 발급
3. 브라우저가 파일 업로드 완료 후 Worker에 완료 통지
4. Worker가 라우팅 규칙으로 목적지 계산
5. Worker가 서버-투-서버로 Google Drive + Synology 업로드 실행
6. 업로드 결과 상태를 저장하고 조회 API로 노출

## 4) 외부 업로드가 가능한 이유
- Worker는 서버 코드이므로 외부 HTTPS API 호출이 가능하다.
- Google/Synology 인증정보는 Worker Secret으로만 저장한다.
- 따라서 브라우저가 직접 외부 저장소 자격증명을 알 필요가 없다.

## 5) Synology 연동 가능 여부 점검 결과 (2026-02-27)

### 5.1 외부 API 노출 확인
- 테스트 URL:
  - `https://izensales.synology.me:62968/webapi/query.cgi?api=SYNO.API.Info&version=1&method=query&query=SYNO.API.Auth,SYNO.FileStation.Upload`
- 응답 요약:
  - `success: true`
  - `SYNO.API.Auth` 확인
  - `SYNO.FileStation.Upload` 확인
- 판정:
  - 외부에서 Synology WebAPI 접근 가능

### 5.2 로그인 API 확인
- 응답 요약:
  - `success: true`
  - `sid` 발급 성공
- 판정:
  - 외부에서 인증 세션 발급 가능

### 5.3 최종 판정
- Synology 자동 업로드 구현 가능 상태
- 남은 실무 확인:
  - 대상 폴더 쓰기 권한 계정 준비
  - 업로드 API(`SYNO.FileStation.Upload`) 실파일 업로드 검증

## 6) Google Drive 연동 가능 여부 점검 기준
- 아래 4개가 모두 가능해야 구현 가능:
1. Google Cloud에서 Drive API 활성화 가능
2. Service Account 생성 가능
3. Service Account JSON 키 발급 가능 (조직 정책 허용)
4. 대상 Drive/Shared Drive 폴더에 Service Account 권한 부여 가능

## 7) 보안 원칙
- 실계정 비밀번호/토큰은 코드/문서/깃에 저장하지 않는다.
- Worker Secret으로만 저장한다.
- 테스트 중 노출된 계정 비밀번호는 즉시 변경한다.

## 8) 환경변수(초안)
- `GOOGLE_SERVICE_ACCOUNT_JSON`
- `GOOGLE_DRIVE_ROOT_FOLDER_ID`
- `SYNOLOGY_BASE_URL`
- `SYNOLOGY_ACCOUNT`
- `SYNOLOGY_PASSWORD`

## 9) 구현 단계 (MVP)
1. 업로드 완료 API 추가: 업로드 메타데이터 + 라우팅 입력 수신
2. 라우팅 규칙 함수 추가: 목적지(Drive folder, Synology path) 계산
3. Google 업로더 추가
4. Synology 업로더 추가
5. 업로드 상태 저장/조회/재시도 API 추가
6. UI에 업로드 상태 표시(성공/부분실패/실패) 추가

## 10) 운영 메모
- Synology는 운영 전용 업로드 계정(`upload_bot`) 분리 권장
- 최소 권한(대상 폴더 read/write만) 적용
- 실패 로그는 원인별(인증/권한/네트워크/경로오류)로 분리해 저장
