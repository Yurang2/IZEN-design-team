# Example Policy

`폴더 구조` 탭의 파일 표시는 두 모드로 나뉩니다.

## 보기 모드
- `예시파일`: 가이드용 샘플 파일명만 표시
- `실제파일`: txt에서 복구한 실제 파일만 표시

두 모드 모두 같은 폴더 뼈대를 사용합니다.  
파일은 폴더 바로 아래에 표시하며, 중간에 `예시파일/실제파일` 하위 폴더를 만들지 않습니다.

## 예시파일 규칙
- 임의 이름 금지
- 현재 `tree.md`, `gdrive.md`, `naming.md` 규칙을 엄격히 따라야 함
- 예시 모드에서는 파일이 들어가는 최종 leaf 폴더마다 최소 1개 이상 샘플이 있어야 함
- 컨테이너 역할의 상위 폴더는 비어 있어도 됨

## 실제파일 규칙
- 원본은 legacy NAS txt 복구 데이터
- 규칙에 따라 rename된 항목은 `원본파일명`을 함께 표시
- 실제파일 모드는 예시파일을 섞지 않음

## 변경 절차
1. 문서 규칙 수정
2. `NasGuideView.tsx` 샘플 트리 수정
3. 필요 시 실제파일 생성 규칙 재생성
4. Notion 이슈 트래커에 결정 배경 기록
