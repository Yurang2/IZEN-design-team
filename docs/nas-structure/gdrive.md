# Google Drive Tree

Google Drive는 예전 `03_LIBRARY` 역할을 이어받은 완성 배포본 보관소입니다.

## 기본 구조
```text
Google Drive
├─ 01_회사소개
├─ 02_카달로그
├─ 03_브로슈어
├─ 04_리플렛
├─ 05_포스터
├─ 06_certificate
├─ 07_배너-사인물
├─ 08_영상
├─ 09_패키지
├─ 10_IFU
├─ 11_기타-배포본
└─ 12__archive
```

## 세부 구조
- `01_회사소개`: `company-profile`, `company-video`
- `02_카달로그`: `I-system`, `T-system`, `R-system`
- `08_영상`: `후기영상`, `홍보영상`, `브랜딩영상`, `모션그래픽`, `제품영상`, `회사소개영상`
- `12__archive`: 구버전 Rev 보관

## 운영 규칙
- Google Drive에는 완성 배포본만 둡니다
- 작업중 소스파일(`v01`, `v02`)은 넣지 않습니다
- Google Drive 파일명은 `Rev01`, `Rev02` 체계를 사용합니다
- 최신 배포본은 카테고리 상위 폴더에 둡니다
- 이전 배포본은 각 카테고리의 `_archive` 또는 `12__archive`에 둡니다

## 현재 가이드 반영 방식
- `폴더 구조` 탭에서도 `Google Drive` 루트가 함께 보입니다
- `구글 드라이브` 탭은 이 구조를 설명용으로 다시 보여줍니다
