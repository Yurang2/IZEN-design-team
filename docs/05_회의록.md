# 05_회의록 페이지 지침

## 1) 목적
- 회의록 탭은 `전사 결과 확인 -> 화자 라벨링 -> Notion 반영`의 3단계를 분리한다.
- 라벨링 전 결과(A/B/C)는 내부 검토용이며, Notion 최종 본문에는 라벨 확정 결과만 기록한다.

## 2) 데이터 저장 원칙
- 별도 중간 DB를 두지 않는다.
- 중간 상태는 Notion transcript row의 속성으로 관리한다.
- 필수 속성:
- `assemblyId`, `status`, `speakerMapJson`, `bodySynced`, `errorMessage`, `textPreview`

## 3) 상태 표시 원칙 (사용자 혼란 방지)
- 화면 표시 상태는 아래처럼 분리한다.
- `전사 진행중`: queued/submitted/processing
- `라벨링 필요`: AssemblyAI 전사 completed 이지만 미매핑 화자 존재 또는 Notion 미반영
- `반영 완료`: 화자 매핑 완료 + Notion 반영 완료(`bodySynced=true`)
- 주의: 단순 전사 완료(completed)를 곧바로 최종 완료로 표기하지 않는다.

## 4) 화자 매핑/반영 규칙
- `매핑 저장`:
- A/B/C -> 실명 매핑을 `speakerMapJson`에 저장만 한다.
- Notion 본문은 이 단계에서 쓰지 않는다.
- `Notion 반영`:
- 매핑 누락 화자가 없어야 실행 가능하다.
- 실행 시 Notion 페이지 본문을 재작성한다.
- `전문`에는 `화자별 발화`만 기록하고, `원문 텍스트` 섹션은 생성하지 않는다.
- 같은 `transcriptId`에 대해 `매핑 저장 + Notion 반영`은 반복 실행 가능하다.
- 이 반복 실행은 AssemblyAI 재전사를 만들지 않으므로 전사 비용을 추가로 발생시키지 않는다.

## 5) 제목 파싱 규칙
- 입력 파일명/제목이 `yymmdd <제목>` 형식이면:
- Notion `날짜` = `20yy-mm-dd`
- Notion 페이지 `이름(제목)` = `<제목>` 부분만 사용
- 형식 미일치 시 전체 문자열을 제목으로 사용하고 날짜는 비워둔다.

## 6) 오디오 첨부 규칙
- Notion 페이지 최상단에 원본 녹음 접근 경로를 둔다.
- 구현 옵션:
- A안(권장): R2 파일을 Notion 파일로 업로드해 첨부(영구 접근성 유리)
- B안: R2 공개 URL 또는 외부 링크 블록 첨부(서명 URL 만료/권한 이슈 점검 필요)
- 보안상 비공개 버킷이면 만료 없는 직접 URL 첨부는 지양한다.

## 7) 최종 반영 조건
- 아래 조건을 모두 충족해야 `반영 완료`로 간주한다.
- 전사 상태 `completed`
- 모든 화자 라벨 매핑 완료
- Notion 본문 반영 성공 (`bodySynced=true`)
